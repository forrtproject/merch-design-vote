# FORRT design vote

A tiny static app for the FORRT community to vote on candidate merch designs. Deployed to GitHub
Pages; votes collect in a Google Sheet via a reusable Apps Script endpoint.

**Live:** _(GitHub Pages URL — enabled in repo Settings ▸ Pages)_

## How it works
- `index.html` / `app.js` / `style.css` — the swipe UI. Loads `manifest.json`, shows each design
  (as an on-product mockup) in **random order**, records **Pass / Keep / Love** plus an optional
  note, and posts every vote to the endpoint. Picks also save on-device (resumable), and each
  voter can download their own picks as CSV.
- `manifest.json` — the list of designs (slug, title, category, product tags). Regenerated by the
  design pipeline.
- `img/` — web-optimised WebP mockups (one per design).
- `config.js` — campaign config: the endpoint URL, the `project` (sheet tab), and titles.
- `apps_script/` — the reusable vote endpoint (`Code.gs`) + `DEPLOY.md`.

## Set up voting (once)
1. Deploy the endpoint — follow `apps_script/DEPLOY.md` and paste the `/exec` URL into `config.js`.
2. Enable Pages — repo **Settings ▸ Pages ▸ Deploy from branch ▸ `main` / root**.
3. Share the Pages URL with the community.

Until the endpoint is set, the app runs in **local mode** (votes saved in the browser and
downloadable), so you can preview it immediately.

## Reusing for another project
The endpoint is campaign-agnostic. Copy this repo (or add a new `config.js` `PROJECT`), drop in a
new `manifest.json` + `img/`, and votes land in a fresh tab of the same workbook.

Designs are community concepts under Creative Commons.
